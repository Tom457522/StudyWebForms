ViewState

状態を PostBack の向こう側へ運ぶ hidden フィールド

本日のゴール

  • ViewState が PostBack をまたいで値を保持する仕組みであることを説明できる
  • ViewState の実体が __VIEWSTATE という hidden フィールドであることを、実際に View Source で確認する
  • ViewState のサイズ・セキュリティ上の注意点を説明できる

前章までの疑問

第3章:Count の値は ViewState["Count"] に入れていた…が、Page は毎回作り直されるはず。どこに保存されているのか?

答え:クライアント(ブラウザ)側の hidden フィールドに保存され、次のリクエストで一緒に送り返されています。

定義:「ViewState」とは

ViewState:
ページの状態をクライアント側に「隠し持たせる」仕組み。Base64 エンコードされた hidden フィールド(__VIEWSTATE)として HTML に埋め込まれ、次のリクエストで自動的にサーバーに返される。

つまり:Page が消えても、状態をクライアント側に預けておいて、次のリクエストで復元する

状態保存の比較

ViewState(Web Forms)

保存場所: クライアント側

実体: __VIEWSTATE hidden field

スコープ: ページ単位

サイズ: レスポンスに含まれる(通信量に影響)

Session(サーバー側)

保存場所: サーバーメモリ

実体: サーバーのセッション辞書

スコープ: ユーザー単位(複数ページ共有)

サイズ: サーバーリソースとしてカウント

Cookie

保存場所: クライアント側

実体: ブラウザの Cookie

スコープ: ドメイン単位

サイズ: 制限あり(4KB程度)

MVC / .NET Core

保存場所: 明示的に選択

実体: Session または Cookie

スコープ: 開発者が管理

サイズ: 明確な制限を意識

ViewState の仕組み

サーバー: ViewState["Memo"]=値
→
HTML の hidden に埋め込む
→
ブラウザに保持される
→
次の送信で一緒に返る
  • ViewState はサーバーのメモリに残っているのではない — 毎回クライアントから送られてくる
  • ページ単位の状態保持であり、他のページ・他ユーザーとは共有されない

コードで確認

04_ViewState.aspx.cs

protected void SaveButton_Click(object sender, EventArgs e)
{
    ViewState["Memo"] = MemoTextBox.Text;
}

protected void ClearButton_Click(object sender, EventArgs e)
{
    ViewState.Remove("Memo");
    MemoTextBox.Text = string.Empty;
}

ViewState.Remove はキーそのものを削除する — 単に null を入れるのとは違う点に注意

実体を見てみる:__VIEWSTATE

ブラウザで「ページのソースを表示」すると…

<input type="hidden" name="__VIEWSTATE" id="__VIEWSTATE" value="/wEPDwUJNjA3NDk4Mzc1D2QWAgIDD2QWAgIBD2QWA...(非常に長い文字列)..." />
  • メモ欄に文字を増やすほど、この文字列(=レスポンスサイズ)が大きくなる
  • Base64 エンコードされているだけで、暗号化はデフォルトでは行われない → 機密情報を入れない

デモで確認するポイント

いつどこを見る何が分かるか
「ViewState に保存」押下後ラベルの表示ViewState["Memo"] に保存された内容が表示される
ページのソースを表示__VIEWSTATE の文字数保存内容が大きいほどレスポンスサイズが増える
「ViewState をクリア」押下後ラベルの表示ViewState.Remove("Memo") によりキーごと削除される

よくある誤解・注意点

  • ViewState はサーバー側に残る仕組みではない(セッションと混同しない)
  • パスワードなど機密情報は ViewState ではなく Session など サーバー側の仕組みで管理する(第5章)
  • 大量のデータを入れすぎるとページの表示・通信が遅くなる

まとめ

  • ViewState の実体は __VIEWSTATE という hidden フィールド(クライアント側に保存)
  • ページ単位の状態保持であり、サイズはレスポンスサイズに直結する
  • 機密情報の保管には向かない
