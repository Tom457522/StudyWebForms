GridView とデータバインド

一覧画面と Repository パターンの入り口

本日のゴール

  • GridView に List<T> をバインドして一覧表示できることを説明できる
  • マークアップ(列定義)とコードビハインド(データ供給)が分離されている意味を理解する
  • Repository パターンにより、画面側を変えずにデータ取得元を差し替えられることを理解する

ここまでの積み上げ

第3章:if (!IsPostBack) パターン / 第6章:入力検証。今回はこれらを組み合わせて「検索できる一覧画面」を作ります。

定義:データバインドと GridView

データバインド(DataBind):
C# の List<T> や DataTable などのデータを、GridView などのコントロールに「束ねる」処理。GridView.DataSource = data; GridView.DataBind(); で実行。
GridView:
テーブル形式で複数件のデータを表示する Web Forms コントロール。データを列ごとに定義でき、ページング・ソート・行の色付けなど多くの機能を備える。
Repository パターン:
データベースアクセスロジック(SQL の実行)を独立したクラスに分離するパターン。画面側(ASPX)はデータベースを直接参照せず、Repository を通じてデータを取得。

マークアップとコードの分離の利点

マークアップ側(.aspx)

役割: 表示の構造・列定義

<asp:BoundField> で列を定義

「どう見せるか」に専念

コードビハインド(.cs)

役割: データ取得・加工・処理

GridView.DataSource = GetData();

「何を表示するか」を管理

利点:
  • デザイナーはマークアップを、プログラマーはコードを独立して修正できる
  • 同じデータ取得ロジックを複数の画面で再利用可能
  • テストが容易(Repository を Mock に差し替え可能)

マークアップとコードの分離

マークアップ側(.aspx)

列の見た目(ヘッダー・並び順)を BoundField で宣言

コードビハインド側(.aspx.cs)

DataSource に何を渡すかを決める(今はList、将来はDB)

App_Code/Course.cs のプロパティ名(Id, Title, Category, DurationMinutes)が、そのまま DataField と対応する

コードで確認②:コードビハインド(後半)

private void BindGrid()
{
    CourseGridView.DataSource = CourseRepository.Search(KeywordTextBox.Text);
    CourseGridView.DataBind();
}

初回表示・検索ボタンのどちらから呼ばれても、最終的にはこの BindGrid() 一本に集約されている

Repository パターン:将来 SQL Server に差し替える

GridView
←
CourseRepository.Search()
←
今:固定List / 将来:SQL Server

今は App_Code/CourseRepository.cs が固定データを返しているだけ。実案件では、この中身だけ SQL Server への SELECT に差し替えれば、画面側(GridView)のコードは無修正で済む — これが第8章につながる設計。

デモで確認するポイント

いつどこを見る何が分かるか
初回表示GridView の行数!IsPostBack のときだけ BindGrid() を呼んでいる
「Web」で検索絞り込まれた行数CourseRepository.Search がキーワードでフィルタしている
キーワードを空にして検索全4件に戻るかSearch が空文字なら全件返す実装になっている

よくある誤解・注意点

  • DataBind() を呼び忘れると、DataSource を設定しても画面に反映されない
  • 検索のたびに全件から絞り込んでいる(大量データでは非効率 → 実案件では DB 側で絞り込む)
  • AutoGenerateColumns="False" によって、表示したい列だけを明示的に制御している

まとめ

  • GridView は DataSource + DataBind() で一覧表示する
  • 列定義(マークアップ)とデータ供給(コードビハインド)は分離されている
  • Repository パターンにより、データ取得元を差し替えても画面側は変更不要
